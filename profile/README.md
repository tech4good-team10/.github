# 이어툰 (EarToon)

> 눈으로만 감상할 수 있었던 웹툰의 그림 정보(시각 데이터)와 말풍선(텍스트)을
> 유기적으로 결합하여 생생하게 **'읽어주는'** 배리어프리 웹툰 플랫폼

**서비스 정의** — AI를 활용해 웹툰의 컷·대사·화자·장면·효과·댓글을 분석하고,
이를 음성·효과음·햅틱 정보로 변환하는 배리어프리 웹툰 접근성 솔루션

**산업적 포지셔닝** — *웹툰 문화산업 위의 배리어프리 감상 인프라*

---

## 1. 해결하고자 하는 사회문제

### ① 시각 중심 포맷으로 인한 K-콘텐츠 소외와 '정서적 단절'

- **글로벌 K-웹툰 시장에서의 철저한 배제** — K-웹툰은 2024년 매출 **2조 2,856억 원**
  (2년 연속 2조 돌파, 전년 대비 +4.4%)의 글로벌 주류 문화가 되었으나, 시각적 연출에
  의존하는 포맷 특성상 시각장애인에게는 접근 자체가 불가능한 영역으로 남아 있습니다.
- **일상 대화 및 또래 문화에서의 문화적 소외** — 인기 웹툰과 원작 기반 OTT 드라마가
  학교·직장·SNS 대화의 핵심 주제가 되는 사회에서, 시각장애인은 대화의 흐름에 동참하지
  못해 깊은 소외감과 정서적 단절을 겪습니다. 『마스크걸』은 OTT 방영 후 원작 웹툰
  조회수가 **121배, 거래액이 166배** 급증했습니다 — *시각장애인도 원작이 궁금합니다.*

### ② 건조한 기계음 TTS가 야기하는 '예술적·감정적 경험의 유실'

- **맥락 없는 기계식 리딩의 한계** — 현재 대형 플랫폼의 대체 텍스트 기술은 말풍선
  글자를 단조로운 기계음으로 차례대로 읽어주는 수준입니다. 별도의 재생 기능도 아니어서
  스크린리더가 이미지 뒤 텍스트를 인식해 읽는 방식에 그칩니다.
- **장르물 특유의 카타르시스 소멸** — 인물의 미묘한 표정 변화, 긴장감을 쥐락펴락하는
  연출 구도, 웹툰의 백미인 효과음(의성어·의태어)이 완전히 누락됩니다. 눈물을 흘리며
  웃는 명장면조차 기계음은 "알았어"라고 건조하게 읽어버려, 웹툰은 '감상하는 예술
  콘텐츠'가 아니라 '조각난 글자를 억지로 해독하는 정보 전달 과정'으로 전락합니다.

## 2. 왜 웹툰인가 — 웹툰을 '본다'는 것의 의미

웹툰을 본다는 것은 그림을 넘어 **댓글·밈·팬덤까지 함께 소비하는 일**입니다.

### A. 경험 가치 — 함께 노는 문화

| 요소 | 내용 |
|---|---|
| 댓글·커뮤니티 | 독자가 작품·작가와 직접 상호작용하는 구조. 베스트 댓글 추천, 복선 추리, 드립 공유가 읽기 과정에 지속적으로 관여 (한국콘텐츠진흥원 『2024 만화·웹툰 이용자 조사』) |
| 밈·트렌드 | 동시대 사회문화 현상을 반영한 인터넷 밈이 패러디 소재로 활발히 순환 (KCI 등재 연구) |
| 팬덤·2차 창작 | 소비하면서 동시에 창작하는 팬층이 팬카페마다 독자적 문화를 형성 (한국애니메이션학회) |
| SNS 확산 | 웹툰 소비 채널로 인스타그램 이용 20.9% — 3년 만에 3배 증가, '인스타툰' 게시물 234만 건+ (『2024 만화산업백서』) |

→ 이어툰이 **댓글 기능을 감상 경험의 일부로 포함**하는 이유입니다. 작품만 들려주는
것은 웹툰 문화의 절반만 전달하는 것입니다.

### B. 기술 필연성 — 왜 단순 TTS로는 안 되고 AI가 필요한가

- **스크롤 리듬** — 웹툰의 스크롤은 작가가 컷 배치와 여백으로 읽는 속도·정지 타이밍을
  설계해 긴장감을 조절하는 연출 도구입니다(크리티컬 씬). 시각장애인에게는 이
  '속도·정지·긴장감' 정보가 통째로 사라지므로, **음성의 pause·내레이션 속도·효과음
  타이밍으로 원작의 리듬을 청각적으로 재설계**해야 합니다. 컷마다 다른 리듬을 판단해야
  하는 작업이라 AI가 필요합니다.
- **카메라 연출** — 웹툰의 컷은 그림이 아니라 카메라입니다. 초점거리·구도·앵글로
  '누구의 눈으로 보고 있는가'를 전달하는데, 정지된 낱컷에는 영화처럼 보완해줄 연속
  프레임·배우의 음성이 없습니다. "A가 있다, B가 웃는다" 식 나열로는 몰입감이
  사라지므로, 컷의 시점까지 읽어내는 vision AI의 의미분석이 필요합니다.

## 3. 시장성 및 성장성

### 성장하는 두 시장의 교차점

이어툰은 **폭발적으로 성장하는 두 시장이 교차하는 지점**에 서 있습니다.

| 시장 | 규모와 전망 |
|---|---|
| K-웹툰 | 2017년 3,799억 → 2020년 1조 돌파 → **2024년 2조 2,856억 원** (문체부·콘진원 『2025 웹툰산업 실태조사』) |
| 글로벌 웹툰 | Grand View Research: 2023년 83억 달러 → **2030년 453억 달러** (CAGR ~27.3%) |
| 글로벌 오디오 콘텐츠 | 골드만삭스: 2019년 220억 달러 → **2030년 753억 달러(약 90조 원)**. 소설 오디오북 중심에서 **웹툰·드라마 원천 IP의 오디오화**로 재편 중 |

### 수요 예측 (국내)

- 국내 등록 시각장애인 약 **25만 명**, 이 중 오디오북 이용자 약 **20만 명(80%)**
  (국립장애인도서관, 더나은미래 2025 인용)
- 시각장애 청년(18~34세) 대상 조사에서 "가장 경험해보고 싶은 온라인 문화예술 분야"
  1위가 **이미지 콘텐츠(웹툰), 70%** — 잠재 수요가 이미 존재하지만 공급이 없는 시장
- 65.4%는 이미 음성·소리의 도움으로 온라인 이미지 콘텐츠를 소비한 경험 보유

### 수익 추정

『2024 만화산업백서』 기준 웹툰 유료 결제자 1인당 월 약 8,000~11,500원
(연 환산 약 10만~14만 원). 이를 대입하면:

| 이용자 규모 | 연간 수익 추정 |
|---|---|
| 3만 명 | 약 30억 원 |
| 10만 명 | 약 100억 원 |
| **20만 명 (오디오북 이용 시각장애인 전체)** | **약 200억 원** |

저시력자까지 확장 시 잠재 이용자 약 40만 명. 여기에 고령자·이동 중 청취 등
"화면을 볼 수 없는 순간"의 비장애인 수요까지 확장 여지가 있습니다.

## 4. 기존 대안과의 차별점

| 기존 대안 | 한계 | 이어툰의 접근 |
|---|---|---|
| 네이버웹툰 AI 대체 텍스트 | 글자만 뽑는 '구조분석'. 스크린리더가 숨겨진 텍스트를 기계음으로 낭독 — 감정·연출·효과음 전부 누락 | 감정까지 읽는 **'의미분석'** — 표정·구도·분위기를 해설·연기·사운드로 변환 |
| 오디오코믹스 등 오디오 드라마 플랫폼 | 성우 더빙 고품질이지만 **작품당 수작업 제작**(펀딩 필요) — 확장 불가, 신작 즉시 대응 불가 | **AI 자동 파이프라인** — 작품 수에 비례하는 수작업 비용 구조를 깨고, 신작을 즉시 변환 |
| 배리어프리 영화 화면해설 | 영상 매체 전용. 웹툰의 컷·스크롤 문법에는 부재 | 웹툰 서사구조(컷·말풍선·화자·스크롤 리듬) **특화 설계** |

```
[ 기존 배리어프리 웹툰 ]  이미지 → OCR 텍스트 추출 → 기계음 TTS 낭독 (딱딱하고 지루함)

[ 이어툰 ]  이미지 → 감정형 멀티 AI 성우 보이스 매핑 + 공간 입체 사운드 + 햅틱 진동
            → 회차당 수 분의 역동적인 '청각 쇼츠' (스낵 컬처의 즐거움 그대로)
```

## 5. 대상 사용자 및 활용 상황

| 타깃 | 범위 | 필요성 |
|---|---|---|
| 1차 | 전맹·실명 이용자 (글로벌 약 4,330만 명, WHO) | 음성·햅틱 기반 변환 필요성이 가장 높음 |
| 2차 | 중증 시각손상 이용자 | 확대·고대비만으로는 컷·말풍선·표정 파악이 어려움 |
| 3차 | 중등도 시각손상·저시력 (실명+중증 합산 글로벌 약 3.38억 명) | 화면 감상 부담 경감, 보조 청취 |
| 확장 | 고령자, 이동 중·화면 피로 이용자 | "눈이 바쁜 순간"의 웹툰 소비 |

**활용 상황**
- **이동 중** — 출퇴근·통학길처럼 스크린리더 조작이 번거로운 상황에서 소리만으로 감상
- **자투리 시간** — 짧은 휴식에 한 회차를 가볍게 청취 (웹툰 특유의 스낵 컬처 경험)
- **신작 공개 직후** — 최신 화를 챙겨 듣고 대화·댓글 문화에 함께 참여

## 6. 핵심 아이디어 및 주요 기능

### 기능 ① 이어보기 — 웹툰 오디오 감상

컷 이미지를 AI가 분석해 **한 편의 오디오 드라마**로 변환합니다.

- **AI 이미지 의미분석** — vision 모델이 컷마다 대사·화자·감정·분위기·소리 단서를
  구조화 데이터로 추출
- **장면 해설** — 대사만으로 알 수 없는 장면 전환·행동·표정을 내레이터가 짚어주는
  음성 화면해설
- **인물별 목소리 캐스팅** — 컷 사이에서 인물을 추적·통합하고, 외형·대사·역할에서
  뽑은 톤 묘사로 500여 개 보이스 중 자동 캐스팅
- **감정 연기** — 표정·몸짓·말풍선 모양(뾰족한 테두리=고함, 점선=속삭임)을 근거로
  감정 프리셋 7종+강도를 골라 '읽기'가 아닌 '연기'로 합성
- **효과음·배경음** — 장면의 소리 단서(빗소리·파도·천둥)와 분위기를 배경 레이어로
- **3D 공간음향 데이터** — 소리의 방향(pan)·거리(depth)를 타임라인에 실어 웹툰 특유의
  카메라 연출을 청각으로 재현
- **햅틱 동기화** — 분노·긴장·속삭임 감정 이벤트에 진동 패턴을 부여하는 다감각 전달

### 기능 ② 댓글 — 웹툰 문화 참여

- 에피소드별 댓글 작성·좋아요·**베스트 댓글 top 3**
- 회차 감상 후 "베스트 댓글 읽어드릴까요?" 흐름으로 실시간 TTS 낭독
- 감상에서 끝나지 않고 웹툰 고유의 **참여 문화(베댓·드립·복선 추리)** 까지 포괄

## 7. 구현 결과

백엔드 파이프라인과 API 서버를 완성하고, 실제 웹툰 컷으로 엔드투엔드 검증을 마쳤습니다.

### 4단계 AI 파이프라인

```
컷 이미지 배열 (순서대로)
  → [1] 컷별 VLM 분석      : 컷당 Claude 1회 호출 — OCR·인물·감정·무드·해설·소리 큐
  → [2] 화 단위 후처리      : 화당 Claude 1회 호출 — 인물 통합 + 보이스 프로필 배정
  → [3] 오디오 생성         : Typecast TTS + 효과음/배경음 매핑 (LLM 없음)
  → [4] 타임라인 조립       : 결정론적 규칙 코드 + 컷별 믹스다운 (LLM 없음)
```

- **인물 동일성 3중 구조** — ①이름 매칭 ②외형 레지스트리 슬라이딩 ③화 단위 병합
  안전망. 컷이 잘려 있어도 "여주=그녀=주인공"을 하나의 인물로 잇습니다.
- **출력** — 컷 번호 기반 오디오 타임라인(JSON) + 컷별 합본 mp3 + 이벤트별 개별 mp3.
  프론트는 컷당 파일 하나를 재생하는 간편 모드와, 문장 단위 탐색·햅틱·공간음향까지
  제어하는 정밀 모드를 선택할 수 있습니다.

### 실측 데모 (실제 웹툰 4컷)

- 처리 시간 약 1분 30초 (분석 ~10초/컷, TTS 포함)
- 감정 연기 — 시큰둥한 "안 아파 ㅋㅋ" → **tonedown**, 절규하는 "패션세포야!!!
  어딨어?!!" → **angry** 고강도로 자동 연기
- 캐스팅 — "다정하고 부드럽게 챙기듯 묻는 목소리" 묘사로 추천 API가 적합 보이스 매칭
  (추천 점수 0.83~0.91)
- 폭풍우 장면에서 "쏴아아 빗소리·파도·천둥" 소리 단서 자동 추출 → 배경음 레이어 매핑

### 운영 도구 — '사람 검수' 전제 설계

- 브라우저 업로드 페이지: 이미지 선택부터 진행 상황 표시까지 한 화면
- **재캐스팅 API**: AI가 성별을 잘못 추정한 캐릭터를 재분석 없이 몇 초 만에 교체
  (바뀐 대사만 재합성하는 내용 해시 캐싱)
- 단계별 중간 산출물(JSON) 저장: 화자 매핑·해설 정확성을 사람이 검수하는
  "골든 타임라인" 루프 — AI 생성 해설의 정확성·유해 표현 검수 체계의 기반

## 8. 사용 기술 및 AI·데이터 활용 방식

| 영역 | 기술 | 활용 방식 |
|---|---|---|
| 이미지 의미분석 | **Claude API (vision)** | 컷당 1회 호출로 OCR+인물+감정+무드 통합 분석. JSON 스키마 고정(구조화 출력)으로 파싱 오류 원천 차단. 직전 컷 요약+인물 레지스트리를 입력에 넣는 맥락 슬라이딩으로 토큰 비용 억제 |
| 인물 통합 | **Claude API (텍스트)** | 화 전체를 한 번에 보고 잘못 갈라진 인물 병합 + 성별·연령대·톤 프로필 배정 |
| 음성 합성 | **Typecast API** | 인물별 보이스, 감정 프리셋 7종+강도(0.5~2.0), 중립 대사는 앞뒤 맥락 기반 스마트 이모션 — 하이브리드 감정 연출 |
| 보이스 캐스팅 | **Typecast 추천 API** | 톤 묘사를 자연어 쿼리로 넣어 캐스팅, 성별·모델 검증 후 필터 폴백 |
| 오디오 믹싱 | **ffmpeg** | 컷 단위 합본(해설+대사 순차, 배경음 루프 믹스) |
| 서버 | **FastAPI (Python)** | 업로드→처리→상태 폴링→타임라인 서빙 + 댓글 API. 무거운 생성은 전부 사전 처리·캐싱, 재생은 정적 파일 서빙 |
| 비용·안정성 설계 | — | LLM 호출은 컷 분석과 인물 통합 **딱 두 군데**로 한정. TTS는 내용 해시 캐싱으로 중복 생성 방지, 단건 실패 시 재시도 후 건너뛰는 방어 로직 |

**접근성 메타데이터의 재사용성** — 파이프라인이 만드는 컷별 장면 설명·화자·감정·효과음
태그는 웹툰 감상에 한정되지 않는 자산입니다: 오디오드라마 대본, 영상화 작품의 화면해설
초안, 다국어 음성 콘텐츠, 장면 검색·추천 메타데이터로 재활용할 수 있습니다.

## 9. 글로벌 확장 및 사업화 방향

### 왜 글로벌인가

- 한국 웹툰 수출은 일본(49.5%)·북미(21.0%)가 70.5%를 차지하며, 대부분 **디지털 전송**
  형태 — 접근성 기능을 플랫폼 차원에서 배포할 수 있는 구조입니다.
- 동일한 이미지 중심 콘텐츠는 각국의 시각장애인에게도 동일한 장벽입니다. 컷 분석·화자
  정보는 **공통 데이터**로 쓰고 장면 설명·음성만 언어별 현지화하면 됩니다.
- **유럽 접근성법(EAA, 2025.6 시행)** 등 접근성이 '선택적 사회공헌'에서 **제품
  기본요건**으로 이동 중 — 접근성 자동화 기술의 산업적 가치가 규제 측면에서도 커집니다.

### 우선 진출 시장

| 순위 | 시장 | 근거 |
|---|---|---|
| 1 | 일본 | 수출 비중 49.5%. 픽코마(2024년 일본 앱 소비자 지출 1위, 연 거래액 1,000억 엔+)·라인망가 등 한국계 플랫폼이 유통 구조 장악. 일본어 OCR·세로쓰기·의성어 지원 시 망가 시장까지 확장 |
| 2 | 북미 | 수출 비중 21.0%. WEBTOON Entertainment(MAU 1.45억)·Disney 협력 등 IP 결합 가속. 오디오북·팟캐스트 소비문화 성숙 |
| 3 | 유럽 | EAA 규제 대응 수요. 다국어 접근성 변환 기술의 활용 가치 |

### 사업화 형태 (B2B 중심)

1. **플랫폼 API** — 컷 분석·화자 추론·장면 설명·감정 태그·접근성 메타데이터를 반환하는
   변환 엔진. 플랫폼 사업자에게 신규 이용자·ESG 성과·규제 대응을 제공
2. **제작자용 편집 도구** — AI 생성 결과를 검수·수정하는 골든 타임라인 워크플로
   (현재 재캐스팅 API·중간 산출물 구조로 기반 구현)
3. **이용자용 감상 인터페이스** — 대사 중심/해설 포함/상세 해설 모드, 캐릭터별 음성,
   햅틱 강도 조절, 베댓 탐색

### 핵심 경쟁우위

- 일반 이미지 설명이 아닌 **웹툰 서사구조 특화** (컷·말풍선·화자·스크롤 리듬 통합 분석)
- 음성+효과음+**햅틱**을 결합한 다감각 전달
- **댓글을 감상 경험의 일부로** 포함 — 콘텐츠를 넘어 문화 접근성
- 자동 생성 결과를 사람이 검수하는 편집 구조 (품질·안전성)
- 플랫폼 단위 대량 변환을 전제로 한 확장성 — 웹툰→망가→그래픽노블→
  **디지털 시각서사 접근성 엔진**

## 10. 기대효과

**이용자 측면**
- 그림·화면 없이도 웹툰의 스토리, 연출, 몰입감을 온전히 경험
- 댓글·밈 등 웹툰 특유의 문화 참여까지 확장

**사회적 측면**
- 시각장애인의 문화 콘텐츠 접근성 확대 — 국내 잠재 수요 20만 명(오디오북 이용
  시각장애인), 글로벌 접근성 필요군 3.38억 명
- "웹툰은 눈으로 보는 것"이라는 전제를 깨는 인식 변화의 계기

**산업적 측면**
- 작품 수에 비례하던 수작업 배리어프리 제작비 구조를 AI 자동화로 개선
- 특정 플랫폼에 한정되지 않는 범용 접근성 인프라로서 시장 공백 공략
  (국내 추정 연 200억 규모, 일본→북미→유럽 확장)
- 접근성 기술 고도화 사례로서 타 콘텐츠(웹소설 삽화, 카드뉴스, 망가, 그래픽노블)로
  확장 가능성
